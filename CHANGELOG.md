## 0.3.0 — 로컬 서버 방식으로 전환

- **로컬 정적 서버 + iframe 임베드**로 렌더 → 실제 브라우저와 동일하게 동작
  (`fetch`, ES 모듈, 절대경로 `/assets`, 웹폰트 등 웹뷰 스킴에서 깨지던 것들 해결)
- **진짜 라이브 리로드**: SSE로 편집/저장 즉시 갱신(전체 웹뷰 새로고침 없이 부드럽게).
  파일 워처로 에디터에 안 열린 파일·외부 변경도 감지
- **저장하지 않은 편집도 미리보기에 반영** (열린 에디터 버퍼 우선 제공)
- 경로 트래버설 방어, 폭넓은 MIME 지원
- 상태바에 미리보기 상태(포트) 표시, **"브라우저에서 열기"** 명령 추가
- 테스트 대폭 확장: 실제 HTTP 서버 통합 테스트 + 확장→서버 관통 테스트 (총 36 케이스)

## 0.2.0 — 안정성 고도화

- **폴더만 열어도** 그 안의 HTML(`index.html` 우선)을 찾아 자동 미리보기
- 흰 화면의 원인이던 중첩 iframe을 제거하고 렌더 HTML을 웹뷰에 직접 주입
- 모든 이벤트 핸들러를 예외 격리(`guard`)하여 오류가 확장 전체로 번지지 않음
- 문서 읽기 실패/삭제 시 크래시 없이 폴백 페이지 표시
- 내용이 바뀌지 않으면 재렌더를 생략해 깜빡임·부하 감소 (수동 새로고침은 강제 렌더)
- 이미 `<base>`가 있는 문서는 중복 주입하지 않음
- 진단 로그를 파일 대신 "HTML Live Viewer" 출력 채널로 이동
- `vscode` 목 기반 단위 테스트 추가 (`npm test`, 16 케이스)
